Changelog - DPV

[v0.2] -14 Jan 2021

Release Notes

v0.2 represents an update to v0.1 in terms of added new concepts in the hierarchy, expanded base vocabulary, and some backwards-incompatible changes.

The core/base vocabulary has new concepts for Risk, Right, and DataSubjectRight. These are associated with PersonalDataHandling using the relations hasRisk and hasRight respectively.

Personal Data Categories has new concepts for BrowsingReferral, PaymentCard, PaymentCardNumber, PaymentCardExpiry, DeviceSoftware, DeviceOperatingSystem, DeviceApplications, AuthenticationHistory

Purposes has new concepts for Marketing, DirectMarketing, Advertising, PersonalisedAdvertising, UsageAnalytics, CommunicationForCustomerCare, LegalCompliance, Payment, SocialMediaMarketing, RegistrationAuthentication

Processing attributes, such as those for indicating systematic monitoring, large scale, etc. have been removed as boolean properties and added as classes. Full list: SystematicMonitoring, EvaluationScoring, MatchingCombining, AutomatedDecisionMaking, LargeScaleProcessing, InnovativeUseOfNewTechnologies. This is so as to enable assocaiting additional information and semantics for concepts, such as specific types, categories, relations, and enable broader use in context of processing. For example, indicating the specific algorithm or process for automated decision making - which could not be indicated using a boolean value. In addition to these, the concept for DataSource has been added. Generic properties added are: hasDataSource, hasAlgorithmicLogic, hasConsequences, hasHumanInvolvement.

Technical and Organisational Measures have new concepts for: SingleSignOn, ImpactAssessment, PIA, DPIA, Consultation, ConsultationWithAuthority. The concept RiskMitigationMeasure and its associated property mitigatesRisk has been added to indicate risk and mitigation. The existing properties for storage, location, duration, and measure implementation have been modified to avoid name duplication. For example, storage as a property and Storage as a concept are equivalent when used as HTML identifiers. To avoid this, the names have been prefixed, which also adds consistency to the vocabulary naming. The properties are: measureImplementedBy, hasStorage, hasLocation, hasDuration.

Entities added as new concepts include LegalEntity, DataSubProcessor, Child, DataProtectionOfficer, Representative, Authority, DataProtectionAuthority, VulnerableDataSubject. Generic properties for expressing information about entities added are: hasName, hasAddress, hasContact, hasIdentifier, hasRepresentative.

[v0.1] - 19 Nov 2020

Release Notes

v0.1 consists of the first release in 18-JUN-2019, and includes subsequent updates till the last update on 19-NOV-2020. This version introduced the 'core' concepts of DPV, or the base vocabulary, and a taxonomical hierarchy of concepts expanding the vocabulary.

The various changes to the vocabulary have all been consolidated under a single version, namely v0.1. Moving forward, the vocabulary will utilise semantic versioning of the form MAJOR.MINOR.TRIVIAL. The v0.x notation denotes the ongoing progress towards a stable and mature vocabulary which will be finalised as v1.
